Read `agent-ops/COACH.md` and follow its protocol exactly to run a
hands-on session now.

- Read `agent-ops/progress.md` to find the current lab/milestone and
  resume there. Labs run in order: lab1 → lab5, then the workshop
  (`agent-ops/workshop/WORKSHOP.md`).
- `$ARGUMENTS`, if given, names a specific lab to run (e.g. `lab3` or
  `workshop`) — only jump if its prerequisites are marked done.
- The user runs the build/execute steps; you verify against each step's
  CHECK line. Theory only on request (each lab's `## Theory` section, and
  `agent-ops/resources/links.md`).
- If any question id sits in `recap_misses`, re-ask it at the start of the
  session before new work, then clear it on a HIT.
- Update `agent-ops/progress.md` at the end of the session.
